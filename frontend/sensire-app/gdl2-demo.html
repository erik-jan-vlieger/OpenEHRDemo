Hoe werkt deze demonstratie?

Kies hieronder een protocol en download het speciale .gdl2 bestand naar je lokale apparaat. Klik daarna op "Open GDL2 Editor". Binnen de Editor (via Upload Project) laad je het bestand in. De GDL2 Editor genereert vervolgens automatisch een testformulier, waarmee je "live" de if/then intelligentie op de achtergrond ziet afvuren ter controle.

Waarom doen we dit zo in de demo?
De interactieve JavaScript webformulieren die je net als Wijkverpleegkundige hebt doorlopen, zijn perfect geoptimaliseerd voor snelheid. Echter, door de logica **ook** wiskundig te verankeren in Guideline Definition Language (GDL2), is alle klinische intelligentie uit het EPD-systeem gesloopt en via Open Source beschikbaar in ons eigen datacentrum. Als patiëntendata wijzigt (bijvoorbeeld bij binnenkomst via een ketenpartner of labwaarde), kan de openEHR server in een batch duizenden patiënten tegelijk polsen tegen onze eigen set .gdl2 bestanden, en signaleren wie acuut aandacht behoeft (transmuraal triageren).
